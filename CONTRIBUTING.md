## 🤝 如何贡献

### 报告问题
如果你发现了bug或有改进建议：

1. **检查现有Issues**：看看是否已经有人提出
2. **创建新Issue**：
   - 使用清晰的标题
   - 详细描述问题或建议
   - 提供复现步骤（如果是bug）
   - 添加相关标签

### 提交代码

#### 1. Fork项目
点击GitHub页面右上角的"Fork"按钮

#### 2. Clone到本地
```bash
git clone https://github.com/mason0510/freelance-guide.git
cd freelance-guide
```

#### 3. 创建分支
```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

#### 4. 进行修改
- 代码风格保持一致
- 添加必要的注释
- 更新相关文档

#### 5. 提交修改
```bash
git add .
git commit -m "feat: 添加XXX功能"
# 或
git commit -m "fix: 修复XXX问题"
```

**提交消息格式**：
- `feat:` 新功能
- `fix:` 修复bug
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 重构
- `test:` 测试相关
- `chore:` 构建/工具相关

#### 6. 推送到你的Fork
```bash
git push origin feature/your-feature-name
```

#### 7. 创建Pull Request
- 访问你的Fork页面
- 点击"Compare & pull request"
- 填写PR描述：
  - 说明修改内容
  - 关联相关Issue
  - 确保CI检查通过

## 📖 代码规范

### Markdown文档
- 使用标准Markdown语法
- 标题层级清晰（不超过3级）
- 代码块指定语言
- 使用表格时对齐
- 链接使用相对路径

### Python脚本
- 遵循PEP 8规范
- 添加docstring
- 使用有意义的变量名
- 添加必要的注释
- 包含错误处理

### Shell脚本
- 使用ShellCheck检查
- 添加注释说明复杂逻辑
- 使用set -e（遇到错误退出）
- 引号包裹变量

## 🎨 项目结构约定

```

├── docs/              # 所有文档
│   ├── international/ # 国际版文档
│   ├── china/        # 中国版文档
│   └── ppt/          # PPT文件
├── templates/        # 可复制的模板
```

**添加新文件时**：
- 放在合适的目录
- 使用清晰的命名
- 更新README.md中的文件列表
- 添加必要的注释
